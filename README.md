# 基于混合启发式蚁群算法的半导体晶圆调度策略

## 项目简介
半导体被称为现代工业的心脏，在晶圆制造的具体生产环节中，70%到80%的工艺步骤都依赖于集群工具的调度效率。本项目针对半导体晶圆制造过程中的复杂调度问题，提出了一种基于混合启发式蚁群算法的调度策略。该策略通过多线程并发执行、动态参数自适应调节、死锁检测与解决机制，在满足严格工艺约束和准时制生产时间限制的前提下，有效优化晶圆加工的整体完工时间，提升系统吞吐量并减少违规事件。

## 赛题背景与挑战
半导体设备调度面临三大核心挑战。第一是多模块协同，系统中包含装卸位、校准模块、真空锁、加工模块、机械手等多个模块需要精确协同工作。第二是复杂工艺约束，包括资源互斥、阀门互斥、禁止超片、清洁机制、真空锁状态转换等多项限制。第三是准时制时间限制，晶圆在模块内驻留时间不得超过15秒，传输时间不得超过30秒。现有调度系统难以同时满足这些约束，本项目通过创新的算法设计有效解决了这些问题。

## 系统组成
调度系统涉及多个功能模块。装卸位共有三个，每个可以放置一个装有25片晶圆的晶圆盒。校准模块用于晶圆加工前的对准操作。真空锁模块分为两类，一类可在真空与大气状态间转换，另一类为固定状态。加工模块共有十个，每个可放置一片晶圆进行加工。机械手包括一个单臂机械手和两个双臂机械手，负责在各模块间搬运晶圆。

## 核心创新
### 多线程并发执行机制
将系统中每片晶圆的加工任务由线程池统一调度和执行，通过模块锁和全局数据锁保护系统资源安全，实现了多线程间的协同调度。这一机制有效平衡了性能与一致性，显著缩短了整体生产时间。每片晶圆的加工流程为取片、校准、转移、加工、冷却、回片六个步骤。

### 混合启发式蚁群算法
将传统蚁群算法与动态调参机制相结合，用于加工模块的路径选择。每只蚂蚁代表一个潜在的调度方案，它们会根据信息素浓度和启发式函数来选择路径。启发函数的定义优先选择能尽快完成加工或当前处于空闲状态的加工模块。算法在搜索过程中对已遍历的路径进行记录与标记，确保后续迭代中不重复选择相同路径，从而减少无效计算并防止陷入局部最优。
信息素更新包括蒸发阶段和增强阶段。增强阶段的信息素奖励引入综合多目标后的得分，评分越小表示性能越好，使得优质路径的信息素浓度更高，形成正反馈闭环。

### 动态参数自适应机制
定义了三个关键性能指标来驱动参数调整。系统负载率通过加工模块、真空锁、机械臂的繁忙程度加权计算，其中加工模块占比最高达到百分之五十，因其对整体效率影响最大。同时还包括死锁频率和准时制违规率两个指标。
根据系统指标动态调整算法参数。高负载时，信息素重要性增大，挥发率增大，算法优先依赖历史信息并加速遗忘过时路径，以快速收敛到可行解。高准时制违规时，信息素重要性增大，确保任务按时完成。高死锁频率时，信息素重要性降低，挥发率上升，算法降低路径依赖并清除无效信息，打破死锁。当处于低负载或正常状态下，算法平衡探索与开发来实现全局优化。当在一段时间内参数没有变化时，引入激励项来提高算法的动态响应能力。

### 晶圆紧急度指标
特别设计了晶圆紧急度指标，当某片晶圆临近准时制时间限制时，其调度优先级会显著提高，确保关键晶圆优先处理。

### 死锁检测与解决机制
对于不可避免的死锁情况，开发了一套基于资源分配图的检测机制。先记录占用资源的晶圆，系统会追踪晶圆接下来需要的路径资源，如果这些资源已被其他晶圆占用，就加入依赖关系，之后用深度优先搜索检查是否形成了死锁环路。若存在死锁环路，则从死锁环中选择牺牲代价小的晶圆进行回滚。
牺牲代价结合加工进度、已完成的步数、资源优先级与准时制紧急度来综合评估。回滚操作包括两步，首先若晶圆处于加工模块，则强制中止加工释放资源，之后选择最近安全的真空锁槽位进行暂时存放，无死锁时再回归加工模块继续加工，最大限度降低对整体调度的影响。

## 实验结果
任务一求解结果中，任务a总处理时间52280秒，准时制违规次数78次；任务b总处理时间28540秒，违规次数5次；任务c总处理时间14590秒，违规次数57次；任务d总处理时间15670秒，违规次数31次。
任务二求解结果中，任务a总处理时间19710秒，违规次数124次；任务b总处理时间38760秒，违规次数123次；任务c总处理时间19120秒，违规次数64次；任务d总处理时间23350秒，违规次数76次。
由于蚁群算法的随机性特征，所有结果为10次运行结果的平均值，标准差控制在正负百分之五以内。

### 参数自适应效果分析
以任务二的d为例进行分析，初期系统负载持续波动上升，之后末期进入较为稳定的自适应调节阶段。准时制违规率先大幅上升后平稳，说明调整有效。系统负载曲线实现了与信息素权重、启发式权重的动态匹配。

### 多目标优化趋势
分别对任务一的d和任务二的d进行分析，两者总时间持续线性上升，说明算法在后期更多倾向于保证其他目标。吞吐量先大幅下降后趋势缓慢平稳，可能是探索范围更广使系统吞吐量增大。准时制违规率逐步上升后趋于稳定，说明系统调度效果持续优化。

## 算法性能分析
主流程涉及每片晶圆按照预设的工艺路径进行加工，复杂度为晶圆总数与晶圆路径长度相乘。所有机械臂每次处理一个动作，最坏情况是每时都在搬运，复杂度为晶圆路径长度与晶圆总数相乘。准时制约束在每次晶圆驻留与传输阶段执行时间检查，检查为常数复杂度，复杂度为晶圆路径长度与晶圆总数相乘。
蚁群算法涉及每片晶圆在多个候选加工模块路径节点的选择、蚂蚁数量以及迭代次数，复杂度为晶圆总数乘加工模块数量乘蚂蚁数量乘迭代次数。死锁检测机制涉及构造等待图和深度优先搜索检查、计算牺牲代价、执行回滚，并且定期检查死锁，因此复杂度为检测次数乘晶圆总数的平方与死锁环中晶圆总数乘路径长度与晶圆回退移动次数之和。
整个算法最坏的时间复杂度为上述各项相加，由于某些量为常数，可简化为晶圆总数的平方与晶圆路径长度乘晶圆总数之和。

## 优化效果对比
相较随机选择加工模块调度，优化后的方案显著缩短了整体调度时间，平均缩短约百分之二十。在整体生产时间、准时制合规性、死锁处理能力与稳定性方面均获得了显著提升。
